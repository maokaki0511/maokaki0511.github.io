SAP Wallet

An event check-in and agenda system designed for an internal conference, connecting online interaction with offline event operations.

Role UX/UI Designer
Team 4 members
Timeline 5 weeks
Focus Workflow UX

Overview

SAP Wallet was designed for an internal event with around 3,000 attendees. The product supported agenda browsing, session check-in, and participation records.

My work focused on clarifying the event-day user flow, improving information architecture, and creating a practical interface that could support both attendees and operational needs.

Problem

At the beginning, the project focused mainly on the check-in function. However, after reviewing the actual event-day scenario, I realized that users would not start from “checking in” itself. They would first need to know where to go, which session to attend, and what action was required at each moment.

Design Direction

I reframed the homepage around the agenda, placing check-in actions closer to the session detail flow. This made the system more aligned with the real offline event journey.

Project image / flow diagram placeholder

Outcome

The system supported more than 20,000 check-ins and over 2,600 logged-in users during the event. Through this project, I learned how important it is to design not only for isolated functions, but also for real-world operational flows.
